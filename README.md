本人24年国一选手，自测示范程序可供参考

本人模板推荐：https://github.com/Z1R343L-D77/lanqiao_mcu

## 项目简介

本项目包含蓝桥杯单片机竞赛（STC15F2K60S2平台）第3-15届的真题程序实现，涵盖省赛（S系列）和国赛（J系列）题目。所有程序均已通过实际测试，可作为学习和参考使用。

## 硬件平台

- **MCU**: STC15F2K60S2
- **主频**: 12MHz（内部时钟）
- **Flash**: 60KB
- **RAM**: 4KB
- **外设**: GPIO、定时器、UART、SPI、I2C、ADC、PWM等

## 开发环境

- **IDE**: Keil uVision 4/5
- **编译器**: C51
- **仿真器**: STC-ISP（用于下载和调试）

## 核心模块说明

每个题目程序都包含以下标准模块：

### 1. 基础驱动模块

- **time.c/h** - 定时器驱动，提供系统时基
- **key.c/h** - 独立按键驱动，支持消抖和长按检测
- **smg.c/h** - 数码管显示驱动，支持动态扫描
- **led8.c/h** - LED灯控制驱动

### 2. 外设驱动模块

- **onewire.c/h** - 单总线驱动，用于DS18B20温度传感器
- **iic.c/h** - I2C总线驱动，用于EEPROM等设备
- **ds1302.c/h** - DS1302实时时钟驱动
- **uart.c/h** - 串口通信驱动
- **wave.c/h** - 波形生成模块（DAC输出）

### 3. 应用控制模块

- **ctrl.c/h** 或 **control.c/h** - 主控制逻辑
- **all.h** - 全局头文件，包含所有模块声明和宏定义

### 4. 主程序

- **main.c** - 主程序入口，包含初始化和主循环

## 使用方法

### 1. 打开项目

使用Keil uVision打开对应题目的`.uvproj`文件：
```
双击 J10_ws_0521/J10.uvproj
```

### 2. 编译程序

在Keil中选择 `Project -> Build Target` 或按 `F7` 键编译项目。

### 3. 下载程序

使用STC-ISP工具：
1. 连接USB转串口模块
2. 选择正确的COM口
3. 选择芯片型号：STC15F2K60S2
4. 打开编译生成的HEX文件
5. 点击"下载程序"按钮
6. 给开发板上电

### 4. 调试程序

- 使用Keil的仿真功能进行软件调试
- 使用STC-ISP的串口助手进行硬件调试
- 通过LED、数码管等外设观察程序运行状态

## 代码规范

### 命名规范

- **文件名**: 小写，使用下划线分隔（如 `key.c`）
- **函数名**: 小写，使用下划线分隔（如 `key_scan()`）
- **变量名**: 小写，使用下划线分隔（如 `led_dat`）
- **宏定义**: 全大写（如 `CTRL`, `LED`）

### 数据类型

```c
typedef unsigned char u8;   // 8位无符号整数
typedef unsigned int u16;  // 16位无符号整数
```

### 硬件抽象

使用宏定义进行硬件抽象，便于移植：
```c
#define CTRL 0xa0
#define LED 0x80
#define WEI 0xc0
#define DUAN 0xe0
#define CLOSE() P2&=0x1f
#define OPEN(a) P2|=a
```

### 存储类型

- `xdata`: 外部RAM，用于大容量数据存储
- `bdata`: 位寻址区，用于位变量
- `volatile`: 易变变量，用于中断和硬件寄存器

## 常见问题

### 1. 编译错误

- 确保已安装STC官方头文件 `STC15F2K60S2.H`
- 检查项目路径中是否包含中文字符

### 2. 下载失败

- 检查USB转串口驱动是否安装
- 确认COM口选择正确
- 检查开发板供电是否正常

### 3. 程序运行异常

- 检查晶振频率设置是否正确
- 确认外设初始化顺序
- 使用LED或串口输出调试信息
